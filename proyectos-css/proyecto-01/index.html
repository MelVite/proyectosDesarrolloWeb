Nuevas propiedades CSS:



transform

Permite modificar el espacio de coordenadas del modelo usando funciones de transformación, los elementos pueden ser trasladados, rotados, escalados o sesgados de acuerdo a los valores establecidos.


Funciones de tranformación


  • matrix

    transform: matrix(a, b, c, d, tx, ty)

    Específica una matriz de transformación 2D compuesta por seis valores a especificar.


  • rotate

    transform: rotate(deg);

    Define una operación de rotación 2D de un elemento, específicando la cantidad de grados (deg) que este rotará en sentido de las manecillas del reloj.

    • rotateX
      transform: rotateX(deg);

      Define una operación de rotación 3D de un elemento en el eje X (derecha o izquierda).

    • rotateY
      transform: rotateY(deg);

      Define una operación de rotación 3D de un elemento en el eje Y (arriba o abajo)<./p>

    • rotateZ
      transform: rotateZ(deg);

      Define una operación de rotación 3D de un elemento en el eje Z(adelante o atrás).


  • scale

    transform: scale(sx[, sy]);

    Especifica una operación de escalado 2D descrita por [sx, sy]. Si sy no es especificado, se asumirá que tanto sx como sy tendrán el mismo valor

    • scaleX
      transform: scaleX(sx);

      Especifica una operación de escalado 2D usando el vector [sx, 1].

    • scaleY
      transform: scaleX(sx);

      Especifica una operación de escalado 2D usando el vector [1, sy].


  • skew

    transform: skew(ax[, ay]);

    Sesga el elemento a lo largo del eje X y Y por los ángulos especificados. Si no se proporciona ay, no se llevará a cabo el sesgo del eje Y.

    • skewX
      transform: skewX(deg);

      Sesga un elemento a lo largo del eje X por el ángulo dado.

    • skewY
      transform: skewY(deg);

      Sesga un elemento a lo largo del eje Y por el ángulo dado.


  • translate

    transform: translate(tx[, ty]);

    Especifica una tanslación 2D dada por el vector [tx, ty] . Si ty no es específicada, se asumirá que su valor es cero.Cada translation-value puede ser un valor de longuitud o un valor de porcentaje.

    • translateX
      transform: translateX(tx);

      Translada un elemento a lo largo del eje X.

    • translateY
      transform: translateY(ty);

      Translada un elemento a lo largo del eje Y.



animation-name

Esta propiedad especifica una lista de animaciones que se deben aplicar al elemento seleccionado. Cada nombre indica un @keyframe esta regla define los valores de las propiedades de la secuencia de animación.



@keyframe

Esta regla permite a los autores controlar los pasos intermedios en una secuencia de animación CSS mediante el establecimiento de keyframes (o puntos de trayectoria) a lo largo de la secuencia de animación que debe ser alcanzado por determinados puntos durante la animación. Esto le da un control más específico sobre los pasos intermedios de la secuencia de animación que se obtiene al dejar que el navegador maneje todo automáticamente.

Para utilizar keyframes, se crea una regla de @keyframes con un nombre que es utilizada por la propiedad animation-name para que coincida con una animación de keyframe a su lista. Cada regla @keyframes contiene una lista de estilo de selectores de keyframe, cada una de los cuales está compuesto de un porcentaje a lo largo de la animación en la que se produce el keyframe así como un bloque que contiene la información de estilo para ese keyframe.

Listas válidas de keyframe

Para obtener una lista de keyframe que sea válida, debe incluir reglas para al menos los tiempos 0% (o desde) y 100% (o hacia) (o sea, los estados inicial y final de la animación). Si ambos desplazamientos de tiempo no se especifican, la declaración keyframe es inválida y no se puede utilizar para la animación.Si se incluyen las propiedades que no se pueden animar en sus reglas de keyframe, serán ignoradas, pero las propiedades admitidas todavía estarán animadas.



animation-duration

Indica la cantidad de tiempo que la animación tarda en completar un ciclo Podemos especificarlo en segundos (usando s) o milisegundos (usando ms). Si no especificamos la unidad, la sentencia no será válida.

El valor por defecto es 0s, que indica que la animación no debe producirse.



animation-iteration-count

Define el numero de veces que un ciclo de animación debe ser ejecutado antes de detenerse

    Valores
  • infinite: La animación se repetirá siempre.
  • number: El número de veces que la animación debe repetir: es 1 por defecto. Los valores negativos no son validos. Puede especificar valores no enteros para ejecutar parte de un ciclo de animación (por ejemplo 0.5 se ejecutara la mitad del ciclo de la animación).
